XML Rules


XML has rules that must be follwed in order to be a vaild xml file. The folllowing rules should be applied when creating XML fiiles:

Root elements

Every element in a XML file must have a root parent that all elements relate to.

XML prolog

The XML prolog is optional, if the prolog is to be used, it must come first in the document. The XML prolog is shown below
<?xml version="1.0" encoding="UTF-8"?>
You should always specify that you are using UFT-8 to avoid errors. Internatinal characters may be used in the prolog.

Closing Tag

All XML elements must have a closing tag: > to specify the ending of an element.

Case Sensativity

Tags in XML are case sensative, this means that <Test> is diffrent from <test>

Indentation

All XML elements must be properly indented. This means that the elements must be in order, shown below is an example of a wrongly indented element:
<a><b>test<a><b>

It should be done as the folllowing:
<a><b>test<b><a>

Entity refrences

If you need to write a arrow:< you must use the URL encoded verson : &lt; becuase if you write in a < it will think you are putting in a end tag.
